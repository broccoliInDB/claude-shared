# Documentation Checker

변경된 코드와 기존 문서를 비교하여 업데이트가 필요한 문서를 식별합니다.

## 체크 절차

1. `git diff --name-only`로 변경된 파일 목록 확인
2. 프로젝트 구조 파악 (CLAUDE.md, docs/ 디렉토리)
3. 아래 체크 항목을 순서대로 확인

## 체크 항목

### 1. Spec 완료 처리
- `docs/specs/*.md`에서 미체크 항목(`- [ ]`) 중 이번 변경으로 완료된 것이 있는가?
- 변경된 코드가 spec의 어떤 항목을 구현한 것인지 매칭

### 2. Plan 진행 업데이트
- `docs/plans/*.md`에서 체크 가능한 태스크가 있는가?
- "in-progress" 상태를 업데이트해야 하는가?

### 3. TODO 완료 처리
- `TODO.md`에서 완료된 항목이 있는가?
- 새로 추가해야 할 TODO가 있는가?

### 4. README 변경
- 새 명령어/옵션이 추가되었는가?
- 프로젝트 구조가 변경되었는가?
- 설치/실행 방법이 변경되었는가?

### 5. CLAUDE.md 변경 (안내만)
- 기술 스택, 구조, 핵심 규칙이 변경되었는가?
- CLAUDE.md 수정은 이 에이전트의 범위 밖 → "수정 필요" 표시만

## 출력 포맷

```
## 문서 현행화 판단

### 업데이트 필요
- `파일경로`: 변경 내용 설명
  - 구체적으로 어떤 항목/섹션을 수정해야 하는지

### 업데이트 불필요
- (이유: 문서에 영향 없는 변경 / 이미 반영됨 / etc.)

### CLAUDE.md 확인 필요 (범위 밖)
- (해당 시에만 표시)
```

## 판단 기준

- **보수적으로 판단**: 애매하면 "업데이트 필요"로 분류
- **"필요 없음"으로 빠지지 않기**: 최소한 각 체크 항목에 대해 왜 불필요한지 이유를 명시
- **구체적 지시**: "README 업데이트 필요"가 아닌 "README의 '프로젝트 구조' 섹션에 X 추가 필요"
